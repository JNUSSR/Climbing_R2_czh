# 上下位机通信系统说明

## 系统架构总览

### 任务分配
- **Task2006** (StartTask2006) - M2006电机控制任务
- **Task3508** (StartTask3508) - M3508电机控制任务  
- **TaskSerial** (StartTaskSerial) - 上位机通信任务

### 消息队列
- **hostCommQueue** - 容量4，用于UART6接收数据包传递

---

## 初始化流程

在 `MX_FREERTOS_Init()` 中按顺序执行：

1. **电机任务初始化**
   - `DJI_M3508_Task_Init()` - 初始化M3508电机PID和CAN通信
   - `DJI_M2006_Task_Init()` - 初始化M2006电机PID和CAN通信

2. **上位机通信初始化**
   - `Host_Communication_Task_Init()` - 初始化UART6 DMA接收

3. **消息队列创建**
   - `hostCommQueueHandle` - 8字节数据包队列

4. **任务创建**
   - 创建三个FreeRTOS任务线程

## 数据流程

### 接收路径（中断 → 任务）

```
上位机发送数据包
    ↓
UART6 DMA接收 (8字节)
    ↓
HAL_UARTEx_RxEventCallback() [drv_uart.c]
    ↓
Host_Communication_UART_Callback() [host_communication_task.cpp]
    ├─ 检查长度 == 8字节
    └─ osMessageQueuePut() ← 仅入队，不做处理
           ↓
Host_Communication_Task() [任务上下文]
    ├─ osMessageQueueGet() ← 阻塞等待
    ├─ Parse_Control_Packet() ← 验证帧头/帧尾/CRC16
    └─ Process_Control_Command() ← 更新目标速度
           ↓
motor_3508_target_speed / motor_2006_target_speed
```

### 控制路径（任务 → 电机）

```
DJI_M3508_Task() / DJI_M2006_Task()
    ↓
Get_Motor_3508_Target_Speed() / Get_Motor_2006_Target_Speed()
    ↓
motor.Set_Target_Omega(target_speed)
    ↓
motor.TIM_PID_PeriodElapsedCallback()
    ↓
CAN总线发送控制指令
```

## 协议格式

### 数据包结构 (8字节)
```
字节0-1: 帧头 0xAA55
字节2:   电机类型 (0x01=M3508, 0x02=M2006)
字节3:   档位 (0x00=停止, 0x01=低速, 0x02=中速, 0x03=高速)
字节4-5: CRC16校验 (对前4字节计算)
字节6:   帧尾 0x0D
字节7:   填充 (结构体对齐)
```

### 档位速度映射
- **GEAR_STOP** (0x00) → 0 rad/s
- **GEAR_LOW** (0x01) → 5 rad/s
- **GEAR_MID** (0x02) → 15 rad/s
- **GEAR_HIGH** (0x03) → 25 rad/s

## 为什么使用消息队列？

### ❌ 不推荐：在中断中直接处理
```cpp
void UART_Callback(uint8_t *data, uint16_t len) {
    Parse_Packet(data);      // 耗时CRC计算
    Process_Command();        // 复杂逻辑
    // ⚠️ 阻塞其他中断，影响实时性
}
```

### ✅ 推荐：消息队列 + 任务处理
```cpp
void UART_Callback(uint8_t *data, uint16_t len) {
    osMessageQueuePut(queue, data, 0, 0);  // 快速入队
    // ✓ 中断快速返回，不影响系统
}

void Task() {
    osMessageQueueGet(queue, &packet, osWaitForever);
    Parse_Packet(&packet);    // 任务上下文处理
    Process_Command(&packet);
}
```

### 优势
1. **中断快速响应** - ISR仅做必要操作（拷贝+入队）
2. **线程安全** - 消息队列自带同步机制
3. **解耦设计** - 接收和处理分离
4. **实时性保证** - 不阻塞高优先级中断
5. **RTOS原生支持** - 无需手动管理标志和缓冲区

## 调试建议

### 检查点
1. UART6是否正确初始化DMA接收
2. 消息队列是否成功创建（非NULL）
3. 数据包CRC16计算是否正确
4. 电机反馈是否正常接收
5. 目标速度是否正确更新

### 常见问题
- **消息队列满** → 增加队列深度或检查任务是否被阻塞
- **CRC校验失败** → 检查上位机CRC算法与STM32是否一致
- **电机无响应** → 检查CAN ID、电机初始化、PID参数
- **UART接收不到** → 检查波特率、DMA配置、中断使能

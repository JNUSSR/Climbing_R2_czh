## 协议概述

### 设计目标

- ✅ **双向通信**：支持上位机控制下位机，下位机反馈状态
- ✅ **多设备支持**：底盘、机械臂、云台、传感器等统一协议
- ✅ **可靠性**：双重CRC校验 + 序列号防丢包
- ✅ **可扩展性**：通过命令ID轻松添加新功能
- ✅ **实时性**：轻量级设计，适合高频控制（100Hz+）
- ✅ **ROS兼容**：适配ros2_control硬件接口

### 协议特点

| 特性 | 说明 |
|------|------|
| 帧头校验 | CRC8校验帧头完整性 |
| 整帧校验 | CRC16校验整个数据包 |
| 序列号 | 0-255循环，检测丢包和重复 |
| 可变长度 | 数据段0-64字节可变 |
| 字节对齐 | 1字节对齐，减少空间浪费 |
| 小端序 | 与ARM Cortex-M一致 |

### 参考标准

本协议参考以下成熟协议设计：
- **MAVLink**：轻量级、命令ID机制
- **DJI SDK**：双重CRC、序列号设计
- **Modbus RTU**：CRC16算法

## 帧格式设计

### 完整帧结构

```
+------+--------+-----+------+--------+------+--------+
| SOF  | LEN(2) | SEQ | CRC8 | CMD_ID | DATA | CRC16  |
| 1B   | 2B     | 1B  | 1B   | 2B     | 0-64B| 2B     |
+------+--------+-----+------+--------+------+--------+
 <------- Header (5B) -------> <--- Payload ---> Tail
```

### 字段说明

| 字段 | 长度 | 类型 | 说明 |
|------|------|------|------|
| **SOF** | 1字节 | uint8 | 起始标志，固定为 `0xA5` |
| **LEN** | 2字节 | uint16 | 数据段长度 = `sizeof(CMD_ID) + sizeof(DATA)` |
| **SEQ** | 1字节 | uint8 | 包序列号，0-255循环 |
| **CRC8** | 1字节 | uint8 | 帧头校验（对LEN+SEQ共4字节计算） |
| **CMD_ID** | 2字节 | uint16 | 命令ID，小端序 |
| **DATA** | 0-64字节 | bytes | 数据段，长度由LEN决定 |
| **CRC16** | 2字节 | uint16 | 整帧校验（从SOF到DATA末尾） |

### 帧长度计算

```
总长度 = 5(Header) + 2(CMD_ID) + DATA_LEN + 2(CRC16)
       = 9 + DATA_LEN (字节)

最小帧: 9字节 (无数据段)
最大帧: 73字节 (64字节数据段)
```

### C语言定义

```c
#pragma pack(push, 1)

#define FRAME_HEADER_SOF        0xA5
#define FRAME_MAX_DATA_LEN      64

// 帧头结构 (5字节)
typedef struct {
    uint8_t  sof;           // 起始字节 0xA5
    uint16_t data_length;   // 数据段长度 (cmd_id + data)
    uint8_t  seq;           // 包序号 (0-255循环)
    uint8_t  crc8;          // 帧头CRC8校验
} FrameHeader;

#pragma pack(pop)
```

## 命令ID定义

### 命令分类

命令ID采用16位编码，按功能模块划分：

```
0x0000 - 0x7FFF: 上位机 → 下位机 (控制命令)
0x8000 - 0xFFFF: 下位机 → 上位机 (反馈数据)
```

### 详细分类表

| 类别 | ID范围 | 说明 |
|------|--------|------|
| **底盘控制** | 0x0100 - 0x01FF | 速度、位置、模式控制 |
| **机械臂控制** | 0x0200 - 0x02FF | 关节位置、速度、末端控制 |
| **云台控制** | 0x0300 - 0x03FF | 俯仰、偏航角度控制 |
| **电机控制** | 0x0400 - 0x04FF | 单电机/电机组控制 |
| **系统命令** | 0x0F00 - 0x0FFF | 复位、配置、心跳 |
| **底盘反馈** | 0x8100 - 0x81FF | 位置、速度、里程计 |
| **机械臂反馈** | 0x8200 - 0x82FF | 关节状态、末端位姿 |
| **传感器数据** | 0x9000 - 0x9FFF | IMU、编码器、力传感器 |
| **系统状态** | 0xF000 - 0xFEFF | 电池、错误、调试信息 |

## 数据包定义

### 1. 底盘控制

#### CMD_CHASSIS_SPEED (0x0101)
**功能**: 控制底盘运动速度  
**数据长度**: 12字节

```c
typedef struct {
    float vx;       // 前进速度 (m/s) [-5.0, 5.0]
    float vy;       // 横向速度 (m/s) [-5.0, 5.0] (仅全向轮)
    float wz;       // 旋转角速度 (rad/s) [-3.14, 3.14]
} ChassisSpeedCmd;
```

**示例**:
```c
ChassisSpeedCmd cmd = {
    .vx = 1.5f,     // 前进1.5m/s
    .vy = 0.0f,     // 不横向移动
    .wz = 0.5f      // 以0.5rad/s旋转
};
```

#### CMD_CHASSIS_POSITION (0x0102)
**功能**: 控制底盘移动到指定位置  
**数据长度**: 12字节

```c
typedef struct {
```

#### CMD_CHASSIS_FEEDBACK (0x8101)
**功能**: 底盘状态反馈

```c
typedef struct {
```

## CRC校验算法

### CRC8 (帧头校验)

**多项式**: 0x31  
**初始值**: 0xFF  
**应用**: 仅对帧头的 `data_length` 和 `seq` 字段（4字节）计算

```c
uint8_t CRC8_Calculate(const uint8_t *data, uint16_t length) {
    uint8_t crc = 0xFF;  // 初始值
    
    for (uint16_t i = 0; i < length; i++) {
        crc ^= data[i];
        for (uint8_t j = 0; j < 8; j++) {
            if (crc & 0x80)
                crc = (crc << 1) ^ 0x31;  // 多项式
            else
                crc <<= 1;
        }
    }
    
    return crc;
}
```

**使用**:
```c
FrameHeader header;
header.sof = 0xA5;
header.data_length = 14;  // 2(cmd_id) + 12(data)
header.seq = 123;
header.crc8 = CRC8_Calculate((uint8_t*)&header.data_length, 4);
```

### CRC16 (整帧校验)

**多项式**: 0xA001 (MODBUS标准)  
**初始值**: 0xFFFF  
**应用**: 从帧头开始到数据段结束的所有字节

```c
uint16_t CRC16_Calculate(const uint8_t *data, uint16_t length) {
    uint16_t crc = 0xFFFF;  // 初始值
    
    for (uint16_t i = 0; i < length; i++) {
        crc ^= data[i];
        for (uint8_t j = 0; j < 8; j++) {
            if (crc & 0x0001)
                crc = (crc >> 1) ^ 0xA001;  // MODBUS多项式
            else
                crc >>= 1;
        }
    }
    
    return crc;
}
```

**使用**:
```c
uint8_t frame[128];
uint16_t frame_len = 5 + 2 + data_len;  // header + cmd_id + data
uint16_t crc16 = CRC16_Calculate(frame, frame_len);
frame[frame_len] = crc16 & 0xFF;        // 低字节
frame[frame_len + 1] = (crc16 >> 8) & 0xFF;  // 高字节
```

## 编解码实现

### STM32端 (C语言)

#### 打包函数

```c
/**
 * @brief 打包数据帧
 * @param cmd_id 命令ID
 * @param data 数据指针
 * @param data_len 数据长度
 * @param seq 序列号
 * @param out_buffer 输出缓冲区 (需至少 9+data_len 字节)
 * @return 打包后的总长度
 */
uint16_t Protocol_PackFrame(uint16_t cmd_id, const uint8_t *data, 
                           uint16_t data_len, uint8_t seq, uint8_t *out_buffer) {
    FrameHeader *header = (FrameHeader*)out_buffer;
    uint16_t payload_len = 2 + data_len;  // cmd_id(2) + data
    
    // 1. 填充帧头
    header->sof = FRAME_HEADER_SOF;  // 0xA5
    header->data_length = payload_len;
    header->seq = seq;
    
    // 2. 计算帧头CRC8 (对 data_length + seq 共4字节)
    header->crc8 = CRC8_Calculate((uint8_t*)&header->data_length, 4);
    
    // 3. 填充命令ID (小端序)
    out_buffer[5] = cmd_id & 0xFF;
    out_buffer[6] = (cmd_id >> 8) & 0xFF;
    
    // 4. 拷贝数据段
    if (data_len > 0 && data != NULL) {
        memcpy(&out_buffer[7], data, data_len);
    }
    
    // 5. 计算整帧CRC16
    uint16_t total_len = 7 + data_len;
    uint16_t crc16 = CRC16_Calculate(out_buffer, total_len);
    out_buffer[total_len] = crc16 & 0xFF;
    out_buffer[total_len + 1] = (crc16 >> 8) & 0xFF;
    
    return total_len + 2;  // 返回总长度
}
```

**使用示例**:
```c
// 发送底盘反馈数据
ChassisFeedback feedback = {
    .timestamp = HAL_GetTick(),
    .position_x = 1.5f,
    .position_y = 2.0f,
    .yaw = 0.785f,
    .velocity_x = 0.5f,
    .velocity_y = 0.0f,
    .angular_vel = 0.1f,
    .status = 0x01
};

uint8_t tx_buffer[128];
static uint8_t seq = 0;

uint16_t frame_len = Protocol_PackFrame(
    CMD_CHASSIS_FEEDBACK,           // 命令ID
    (uint8_t*)&feedback,            // 数据
    sizeof(ChassisFeedback),        // 数据长度
    seq++,                          // 序列号自增
    tx_buffer                       // 输出缓冲区
);

// 通过UART发送
HAL_UART_Transmit_DMA(&huart6, tx_buffer, frame_len);
```

#### 解析函数

```c
/**
 * @brief 解析数据帧
 * @param buffer 接收缓冲区
 * @param length 接收长度
 * @param cmd_id 输出命令ID指针
 * @param data 输出数据指针
 * @param data_len 输出数据长度指针
 * @return 1=成功, 0=失败
 */
uint8_t Protocol_ParseFrame(const uint8_t *buffer, uint16_t length,
                           uint16_t *cmd_id, uint8_t **data, uint16_t *data_len) {
    // 1. 检查最小长度
    if (length < 9) return 0;  // 最小帧: 5+2+2=9字节
    
    // 2. 检查帧头标志
    if (buffer[0] != FRAME_HEADER_SOF) return 0;
    
    FrameHeader *header = (FrameHeader*)buffer;
    
    // 3. 验证帧头CRC8
    uint8_t crc8_calc = CRC8_Calculate((uint8_t*)&header->data_length, 4);
    if (crc8_calc != header->crc8) return 0;
    
    // 4. 检查长度是否匹配
    uint16_t expected_len = 5 + header->data_length + 2;  // header + payload + crc16
    if (length != expected_len) return 0;
    
    // 5. 验证整帧CRC16
    uint16_t crc16_calc = CRC16_Calculate(buffer, length - 2);
    uint16_t crc16_recv = buffer[length - 2] | (buffer[length - 1] << 8);
    if (crc16_calc != crc16_recv) return 0;
    
    // 6. 提取命令ID (小端序)
    *cmd_id = buffer[5] | (buffer[6] << 8);
    
    // 7. 提取数据段
    *data_len = header->data_length - 2;  // 减去cmd_id的2字节
    if (*data_len > 0) {
        *data = (uint8_t*)&buffer[7];
    } else {
        *data = NULL;
    }
    
    return 1;  // 解析成功
}
```

**使用示例**:
```c
// UART接收回调
void UART_RxCallback(uint8_t *rx_buffer, uint16_t length) {
    uint16_t cmd_id;
    uint8_t *data_ptr;
    uint16_t data_len;
    
    // 解析数据帧
    if (Protocol_ParseFrame(rx_buffer, length, &cmd_id, &data_ptr, &data_len)) {
        // 根据命令ID分发处理
        switch (cmd_id) {
            case CMD_CHASSIS_SPEED: {
                if (data_len == sizeof(ChassisSpeedCmd)) {
                    ChassisSpeedCmd *cmd = (ChassisSpeedCmd*)data_ptr;
                    Chassis_SetSpeed(cmd->vx, cmd->vy, cmd->wz);
                }
                break;
            }
            
            case CMD_ARM_JOINT_POS: {
                if (data_len == sizeof(ArmJointPosCmd)) {
                    ArmJointPosCmd *cmd = (ArmJointPosCmd*)data_ptr;
                    Arm_SetJointPositions(cmd->positions, cmd->joint_num);
                }
                break;
            }
            
            case CMD_MOTOR_CONTROL: {
                if (data_len == sizeof(MotorControlCmd)) {
                    MotorControlCmd *cmd = (MotorControlCmd*)data_ptr;
                    Motor_SetControl(cmd->motor_id, cmd->control_mode, 
                                   cmd->target_value);
                }
                break;
            }
```

#### 打包函数

```python
import struct

def crc8_calculate(data: bytes) -> int:
    """计算CRC8"""
    crc = 0xFF
    for byte in data:
        crc ^= byte
        for _ in range(8):
            if crc & 0x80:
                crc = ((crc << 1) ^ 0x31) & 0xFF
            else:
                crc = (crc << 1) & 0xFF
    return crc

def crc16_calculate(data: bytes) -> int:
    """计算CRC16 (MODBUS)"""
    crc = 0xFFFF
    for byte in data:
        crc ^= byte
        for _ in range(8):
            if crc & 0x0001:
                crc = (crc >> 1) ^ 0xA001
            else:
                crc >>= 1
    return crc & 0xFFFF

def pack_frame(cmd_id: int, data: bytes, seq: int) -> bytes:
    """
    打包数据帧
    
    Args:
        cmd_id: 命令ID (0-65535)
        data: 数据段 (bytes)
        seq: 序列号 (0-255)
    
    Returns:
        完整的帧数据 (bytes)
    """
    SOF = 0xA5
    data_len = len(data)
    payload_len = 2 + data_len  # cmd_id(2) + data
    
    # 1. 构建帧头 (不含CRC8)
    header_partial = struct.pack('<HB', payload_len, seq)
    
    # 2. 计算帧头CRC8
    crc8 = crc8_calculate(header_partial)
    
    # 3. 完整帧头
    header = struct.pack('<BHBB', SOF, payload_len, seq, crc8)
    
    # 4. 命令ID + 数据段
    payload = struct.pack('<H', cmd_id) + data
    
    # 5. 计算整帧CRC16
    frame_without_crc = header + payload
    crc16 = crc16_calculate(frame_without_crc)
    
    # 6. 完整帧
    frame = frame_without_crc + struct.pack('<H', crc16)
    
    return frame

# 使用示例
def pack_chassis_speed(vx: float, vy: float, wz: float, seq: int) -> bytes:
    """打包底盘速度控制命令"""
    CMD_CHASSIS_SPEED = 0x0101
    data = struct.pack('<fff', vx, vy, wz)
    return pack_frame(CMD_CHASSIS_SPEED, data, seq)

def pack_motor_control(motor_id: int, mode: int, value: float, seq: int) -> bytes:
    """打包电机控制命令"""
    CMD_MOTOR_CONTROL = 0x0401
    data = struct.pack('<BBf', motor_id, mode, value)
    return pack_frame(CMD_MOTOR_CONTROL, data, seq)
```

#### 解析函数

```python
def parse_frame(buffer: bytes) -> tuple:
    """
    解析数据帧
    
    Args:
        buffer: 接收到的数据 (bytes)
    
    Returns:
        (success, cmd_id, data, seq)
        success: bool - 解析是否成功
        cmd_id: int - 命令ID
        data: bytes - 数据段
        seq: int - 序列号
    """
    # 1. 检查最小长度
    if len(buffer) < 9:
        return (False, 0, b'', 0)
    
    # 2. 检查帧头标志
    if buffer[0] != 0xA5:
        return (False, 0, b'', 0)
    
    # 3. 解析帧头
    sof, payload_len, seq, crc8_recv = struct.unpack('<BHBB', buffer[:5])
    
    # 4. 验证帧头CRC8
    crc8_calc = crc8_calculate(struct.pack('<HB', payload_len, seq))
    if crc8_calc != crc8_recv:
        return (False, 0, b'', 0)
    
    # 5. 检查长度
    expected_len = 5 + payload_len + 2
    if len(buffer) != expected_len:
        return (False, 0, b'', 0)
    
    # 6. 验证整帧CRC16
    crc16_calc = crc16_calculate(buffer[:-2])
    crc16_recv = struct.unpack('<H', buffer[-2:])[0]
    if crc16_calc != crc16_recv:
        return (False, 0, b'', 0)
    
    # 7. 提取命令ID和数据
    cmd_id = struct.unpack('<H', buffer[5:7])[0]
    data = buffer[7:-2]
    
    return (True, cmd_id, data, seq)

# 使用示例
def parse_chassis_feedback(data: bytes) -> dict:
    """解析底盘反馈数据"""
    if len(data) != 33:
        return None
    
    timestamp, pos_x, pos_y, yaw, vel_x, vel_y, ang_vel, status = \
        struct.unpack('<Iffffffb', data)
    
    return {
        'timestamp': timestamp,
        'position': (pos_x, pos_y, yaw),
        'velocity': (vel_x, vel_y, ang_vel),
        'status': status
    }
```

```python
# serial_bridge_node.py
import rclpy
from rclpy.node import Node
from geometry_msgs.msg import Twist
from nav_msgs.msg import Odometry
import serial

CMD_CHASSIS_SPEED = 0x0101
CMD_CHASSIS_FEEDBACK = 0x8101
CMD_HEARTBEAT = 0x0FFF

class SerialBridgeNode(Node):
    def __init__(self):
        super().__init__('serial_bridge_node')
        
        # 串口
        self.serial = serial.Serial('/dev/ttyUSB0', 921600, timeout=0.01)
        
        # 序列号
        self.tx_seq = 0
        
        # 订阅控制命令
        self.create_subscription(Twist, '/cmd_vel', self.cmd_vel_callback, 10)
        
        # 发布反馈
        self.odom_pub = self.create_publisher(Odometry, '/odom', 10)
        
        # 定时器
        self.create_timer(0.01, self.read_feedback)    # 100Hz读取
        self.create_timer(0.1, self.send_heartbeat)    # 10Hz心跳
    
    def cmd_vel_callback(self, msg):
        """处理速度控制命令"""
        frame = pack_chassis_speed(
            msg.linear.x, 
            msg.linear.y, 
            msg.angular.z,
            self.tx_seq
        )
        self.tx_seq = (self.tx_seq + 1) % 256
        self.serial.write(frame)
        self.get_logger().debug(f'Sent chassis speed: {msg.linear.x:.2f}m/s')
    
    def read_feedback(self):
        """读取STM32反馈数据"""
        if self.serial.in_waiting > 0:
            data = self.serial.read(self.serial.in_waiting)
            
            # 查找帧头并解析
            idx = data.find(b'\xA5')
            if idx >= 0:
                success, cmd_id, payload, seq = parse_frame(data[idx:])
                
                if success:
                    if cmd_id == CMD_CHASSIS_FEEDBACK:
                        feedback = parse_chassis_feedback(payload)
                        if feedback:
                            self.publish_odom(feedback)
    
    def send_heartbeat(self):
        """发送心跳包"""
        import time
        timestamp = int(time.time() * 1000) & 0xFFFFFFFF
        data = struct.pack('<IB', timestamp, 0xAA)
        frame = pack_frame(CMD_HEARTBEAT, data, self.tx_seq)
        self.tx_seq = (self.tx_seq + 1) % 256
        self.serial.write(frame)
    
    def publish_odom(self, feedback):
        """发布里程计数据"""
        odom = Odometry()
        odom.header.stamp = self.get_clock().now().to_msg()
        odom.header.frame_id = 'odom'
        odom.child_frame_id = 'base_link'
        
        odom.pose.pose.position.x = feedback['position'][0]
        odom.pose.pose.position.y = feedback['position'][1]
        # ... 设置其他字段
        
        self.odom_pub.publish(odom)

def main():
    rclpy.init()
    node = SerialBridgeNode()
    rclpy.spin(node)
    node.serial.close()
    rclpy.shutdown()
```

```cpp
class MyRobotHardware : public hardware_interface::SystemInterface {
public:
    CallbackReturn on_init(const hardware_interface::HardwareInfo & info) override {
        // 初始化串口
        std::string port = info_.hardware_parameters["serial_port"];
        int baudrate = std::stoi(info_.hardware_parameters["baud_rate"]);

    }

        std::vector<StateInterface> state_interfaces;
        for (size_t i = 0; i < info_.joints.size(); ++i) {
            state_interfaces.emplace_back(
                info_.joints[i].name, "position", &hw_positions_[i]);
            state_interfaces.emplace_back(
                info_.joints[i].name, "velocity", &hw_velocities_[i]);
        }
        return state_interfaces;
    }
    
    std::vector<CommandInterface> export_command_interfaces() override {
        std::vector<CommandInterface> command_interfaces;
        for (size_t i = 0; i < info_.joints.size(); ++i) {
            command_interfaces.emplace_back(
                info_.joints[i].name, "position", &hw_commands_[i]);
        }
        return command_interfaces;
    }
    
    return_type read(const rclcpp::Time & time, const rclcpp::Duration & period) override {
        // 从串口读取反馈数据
        if (serial_->available()) {
            std::vector<uint8_t> buffer;
            serial_->read(buffer, serial_->available());
            
            // 解析帧
            uint16_t cmd_id;
            std::vector<uint8_t> data;
            if (parse_frame(buffer, cmd_id, data)) {
                if (cmd_id == CMD_ARM_FEEDBACK) {
                    // 更新关节状态
                    update_joint_states(data);
                }
            }
        }
        return return_type::OK;
    }
    
    return_type write(const rclcpp::Time & time, const rclcpp::Duration & period) override {
        // 将命令发送到STM32
        std::vector<float> positions(hw_commands_.begin(), hw_commands_.end());
        auto frame = pack_arm_joint_pos(positions, tx_seq_++);
        serial_->write(frame);
        
        return return_type::OK;
    }

    std::vector<double> hw_velocities_;
    std::vector<double> hw_commands_;
```

## 性能与优化

### 传输频率建议

| 数据类型 | 推荐频率 | 数据量 | 说明 |
|----------|----------|--------|------|
| 底盘速度控制 | 100Hz | 21字节/帧 | 实时控制 |
| 机械臂关节控制 | 100Hz | 34字节/帧 | 轨迹跟踪 |
| 底盘反馈 | 50Hz | 42字节/帧 | 状态反馈 |
| 机械臂反馈 | 50Hz | 86字节/帧 | 状态反馈 |
| IMU数据 | 100Hz | 53字节/帧 | 高频传感器 |
| 系统状态 | 10Hz | 25字节/帧 | 低频监控 |
| 心跳包 | 10Hz | 14字节/帧 | 保活检测 |

### 波特率选择

```
总带宽需求 = Σ(频率 × 数据量 × 10位/字节)

示例：
底盘控制: 100Hz × 21B × 10 = 21,000 bps
底盘反馈: 50Hz × 42B × 10 = 21,000 bps
机械臂控制: 100Hz × 34B × 10 = 34,000 bps
机械臂反馈: 50Hz × 86B × 10 = 43,000 bps
IMU数据: 100Hz × 53B × 10 = 53,000 bps
其他: 约 10,000 bps
----------------------------------------
总计: 约 182,000 bps

推荐波特率: 460800 或 921600 (留50%余量)
```

### 数据压缩建议

对于高频数据，可以优化：

1. **差分编码**：只传输变化量
```c
typedef struct {
    int16_t delta_x;    // 位置变化 (mm) 代替 float
    int16_t delta_y;
    int16_t delta_yaw;  // 角度变化 (0.01度)
} ChassisOdomDelta;  // 从33字节减少到6字节
```

2. **状态标志位压缩**：
```c

// bit0-1: joint1 (00=正常, 01=警告, 10=错误)
// bit2-3: joint2
// ...
```

3. **采样率动态调整**：

## FAQ

### Q1: 为什么用双重CRC校验？
**A**: CRC8快速验证帧头完整性，避免解析错误数据；CRC16保证整帧数据完整性。这是DJI等成熟协议的标准做法。

### Q2: 序列号有什么用？
**A**: 
- 检测丢包：序列号不连续说明丢包
- 检测重复包：重复序列号说明重传
- 调试追踪：便于定位问题帧

### Q3: 最大数据段64字节够用吗？
**A**: 对于大多数机器人控制足够。如需更大（如图像数据），可：
- 分包传输
- 增加MAX_DATA_LEN（需修改双方代码）
- 使用其他协议（如ROS2 DDS）

### Q4: 小端序还是大端序？
**A**: 本协议使用**小端序**（Little-Endian），与ARM Cortex-M一致。网络传输通常用大端序，但机器人内部通信优先考虑MCU效率。

### Q5: 如何处理半包/粘包？
**A**: 
```c
// 维护接收缓冲区
static uint8_t rx_fifo[512];
static uint16_t rx_index = 0;

void UART_Receive(uint8_t *data, uint16_t len) {
    memcpy(&rx_fifo[rx_index], data, len);
    rx_index += len;
    
    // 查找帧头
    while (rx_index >= 9) {
        uint8_t *sof = memchr(rx_fifo, 0xA5, rx_index);
        if (!sof) {
            rx_index = 0;  // 无帧头，清空
            break;
        }
        
        uint16_t offset = sof - rx_fifo;
        uint16_t frame_len = /* 从帧头解析长度 */;
        
        if (rx_index >= offset + frame_len) {
            // 完整帧，解析
            Protocol_ParseFrame(sof, frame_len, ...);
            
            // 移除已处理数据
            memmove(rx_fifo, sof + frame_len, rx_index - offset - frame_len);
            rx_index -= (offset + frame_len);
        } else {
            break;  // 等待更多数据
        }
    }
}
```
